Part1.JSP环境搭建及入门

  • 静态网页： html、cs、js、jquery

  • 动态网页：百度搜索房价、天气（会随着时间、地点、用户操作改变而改变）

  • CS架构： Client Server架构

  • CS架构不足：

    • 1.如果软件升级，所有软件都需要升级
    • 2.维护麻烦：需要维护每一台客户端软件
    • 3.每一台客户端都需要安装客户端软件

  • CS架构优势：

    • 软件较为美观

  • BS架构： Browser Server架构

  • BS架构优势：

      1. 客户端可以通过浏览器直接访问服务端
      1. 降低维护成本

Tomcat目录结构及功能

将Tomcat从官网https://tomcat.apache.org/whichversion.html
解压后先看文件结构：
bin: 可执行文件:(Windows:startup.bat/shutdown.bat)
Mac/Linux:startup.sh/shutdown.sh)
conf:配置文件（server.xml）用于配置Tomcat
lib：Tomcat依赖的jar包
log：日志文件（记录出错等信息）
temp：临时文件
webapps：可执行文件（将我们开发的项目 放入该目录）
work：存放由jsp翻译成的Java，以及编译成的class文件
jsp -> java ->class
